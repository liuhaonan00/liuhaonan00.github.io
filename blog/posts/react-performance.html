React 性能优化实战指南

React 应用在规模增长后，性能问题往往会逐渐浮现。本文系统梳理常见的优化策略，帮助你构建更快的 React 应用。

1. 减少不必要的重新渲染

React.memo

对于纯展示组件，使用 React.memo 可以在 props 没有变化时跳过重新渲染：

const UserCard = React.memo(({ name, avatar }) => {
  return (
    <div className="user-card">
      <img src={avatar} alt={name} />
      <span>{name}</span>
    </div>
  );
});

useMemo 和 useCallback

useMemo 用于缓存计算结果，useCallback 用于缓存函数引用，避免子组件因引用变化而重新渲染：

const filteredList = useMemo(
  () => items.filter(item => item.status === 'active'),
  [items]
);

const handleClick = useCallback(
  (id) => dispatch({ type: 'SELECT', id }),
  [dispatch]
);

注意：不要滥用 memoization。只在确认存在性能瓶颈时才使用，否则反而增加内存开销和代码复杂度。

2. 代码分割（Code Splitting）

使用 React.lazy 和 Suspense 实现路由级别的懒加载：

const Dashboard = React.lazy(() => import('./pages/Dashboard'));
const Settings = React.lazy(() => import('./pages/Settings'));

function App() {
  return (
    <Suspense fallback={<Loading />}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/settings" element={<Settings />} />
      </Routes>
    </Suspense>
  );
}

3. 虚拟化长列表

渲染上万条数据时，使用虚拟化库只渲染可视区域内的元素：

import { FixedSizeList } from 'react-window';

<FixedSizeList
  height={600}
  itemCount={10000}
  itemSize={50}
  width="100%"
>
  {({ index, style }) => (
    <div style={style}>Row {index}</div>
  )}
</FixedSizeList>

4. 状态管理优化

  • 将频繁更新的状态和不常变化的状态分离到不同的 Context 中
  • 使用 useReducer 替代复杂的 useState 组合
  • 考虑使用 Zustand、Jotai 等轻量状态库减少不必要的重渲染

总结

性能优化的首要原则是先测量，再优化。使用 React DevTools Profiler 和 Chrome Performance 面板定位真正的瓶颈，避免过早优化。好的架构设计往往比微观优化更有效。
