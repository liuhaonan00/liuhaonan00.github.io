Docker 容器化最佳实践

Docker 已经成为现代应用部署的标配。本文总结了在生产环境中使用 Docker 的一些最佳实践和经验教训。

1. 编写高效的 Dockerfile

使用多阶段构建

多阶段构建可以显著减小最终镜像的体积：

# 构建阶段
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

# 运行阶段
FROM node:20-alpine
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
EXPOSE 3000
CMD ["node", "dist/server.js"]

合理利用缓存层

将变化频率低的指令放在前面，变化频率高的放在后面。例如先复制 package.json 安装依赖，再复制源代码，这样代码修改不会触发依赖重新安装。

2. 镜像安全

  • 使用官方基础镜像，优先选择 alpine 等精简版
  • 不要在镜像中包含密钥、证书等敏感信息
  • 使用非 root 用户运行容器
  • 定期扫描镜像漏洞：docker scout cves <image>
# 使用非 root 用户
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser

3. Docker Compose 实践

使用 docker-compose.yml 管理多容器应用，利用 profiles 区分开发和生产环境：

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres:16-alpine
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  pgdata:

4. CI/CD 集成

在 CI 流水线中集成 Docker 构建和推送：

  • 使用 BuildKit 加速构建：DOCKER_BUILDKIT=1 docker build .
  • 利用 GitHub Actions 的缓存机制避免重复拉取基础镜像
  • 为镜像打上语义化版本标签和 Git SHA 标签

总结

容器化不仅仅是把应用打包成镜像。良好的 Dockerfile 编写习惯、安全意识和 CI/CD 流程整合，才能真正发挥 Docker 在生产环境中的价值。
